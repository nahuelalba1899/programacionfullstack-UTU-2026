# PROGRAMACION FULL STACK - UTU 2026

## Estructura del repositorio

Este repositorio está organizado para mantener ordenados los materiales del curso, las entregas del proyecto y la documentación. Cada carpeta tiene una función específica para facilitar el trabajo y la búsqueda de archivos.

### CLASES

En esta carpeta se guardan los materiales vistos durante las clases. Puede incluir apuntes, ejemplos de código, ejercicios y presentaciones utilizadas en el curso.

### DOCUMENTACION

Contiene toda la documentación relacionada con los proyectos, como explicaciones del sistema, diagramas, pseudocódigo, instrucciones y cualquier información técnica necesaria para comprender el desarrollo.

### ENTREGA FINAL

Aquí se encuentra la versión final del proyecto que será entregada. Incluye los archivos definitivos del trabajo ya terminado.

### ENTREGA NUMERO 1 PROYECTO

Contiene los archivos correspondientes a la primera entrega del proyecto. Sirve para mostrar el primer avance realizado.

### ENTREGA NUMERO 2 PROYECTO

Incluye los archivos de la segunda entrega del proyecto, donde se agregan mejoras y nuevas funcionalidades respecto a la primera entrega.

### IMAGENES PARA USAR

Esta carpeta almacena recursos gráficos que se utilizan en los proyectos, como logos, iconos, imágenes o elementos visuales.

### INTEGRANTES

Aquí se encuentra la información sobre las personas que participan en el proyecto, como los nombres de los integrantes y posibles roles dentro del equipo.

### PLANIFICACION PROYECTO

Contiene la planificación del proyecto, como cronogramas, objetivos, organización de tareas y planificación del desarrollo.

### PROYECTOS

En esta carpeta se guardan los distintos proyectos realizados durante el curso.

### TRABAJOS

Incluye trabajos prácticos o actividades realizadas como parte de las tareas del curso.

### README.md

Este archivo describe el repositorio, su objetivo y la estructura de carpetas para que cualquier persona pueda entender cómo está organizado el proyecto.

## Motivo de la organización

Se eligió esta estructura para mantener el repositorio ordenado y claro. Separar las clases, los trabajos, los proyectos y las entregas permite encontrar los archivos fácilmente, mostrar el progreso del proyecto y mantener una organización similar a la utilizada en proyectos reales de desarrollo de software.
